Coherent Point Drift

Created by Pete Gadomski, based on work by Andriy Myronenko.

https://github.com/gadomski/cpd

Overview

  • Point set registration algorithm
  • Supports rigid, affine, and non-rigid transformations
  • C++ library using modern linear algebra packages

Summary of operation

  • Treat "source" points as a Gaussian Mixture Model
  • Use Expectation-Maximization (EM) to iteratively transform "target" points and bring them closer to source points
  • Choice of transformation functions:
    • Rigid
    • Affine
    • Non-rigid

Future work

  • Improve non-rigid registration performance
    • Reasonably performant (2-10 minutes) on datasets up to 100,000 points
    • Large datasets could be "chipped", or broken up into pieces
    • One dependency (figtree, for Fast Gauss Transforms) is our main bottleneck
  • Multi-threading support
    • "Chipped" non-rigid transformations could take advantage of multiple cores
    • figtree is not thread-safe

https://github.com/gadomski/cpd/issues
